% ==================== 第一章 需求分析 ====================
\section{需求分析}

\subsection{系统目标}

本项目面向机房、实验室等教学管理场景，设计并实现一套端-边-云协同安全监测系统。系统以边端摄像头和YOLO-Pose模型作为实时感知入口，在边缘侧完成视频采集、人体检测、人体关键点识别、姿态规则判定和初步事件生成；在云端完成事件持久化、知识库检索、大模型综合分析、日志查询、隐患扫描和日报生成；在前端通过边端工作台与云端控制台展示实时画面、检测结果、事件列表、智能体问答、知识库和报告。

系统设计目标不是单纯完成目标检测，而是形成“边端实时判断、云端复杂复核、管理端可视化处置”的完整业务闭环。边端优先承担实时性要求高的处理任务，云端承担需要历史上下文、知识库和大模型推理的复杂任务，从而兼顾实时性、稳定性和智能分析能力。

\subsection{非功能需求}

\begin{itemize}
\item \textbf{实时性}: 边端视频推送默认限制为24fps，检测任务按\verb|EDGE_SKIP_FRAMES|跳帧执行，避免每帧推理造成CPU压力。
\item \textbf{边端自治}: 云端不可用时，边端仍能完成检测、姿态分析、事件生成和本地状态更新。
\item \textbf{异步解耦}: 摄像头采集、检测推理、视频推送、状态广播和云端同步拆分为不同线程或异步任务。
\item \textbf{可扩展性}: LLM、搜索、知识库、数据库和推理后端均通过配置选择，便于在离线演示和真实服务之间切换。
\item \textbf{可部署性}: 通过Docker Compose编排数据库、云端API、边端API、双前端和可选边端采集器。
\end{itemize}

\subsection{数据流设计}

\begin{enumerate}
\item \textbf{视频流}: 摄像头原始帧 $\rightarrow$ 缩放与编码 $\rightarrow$ WebRTC $\rightarrow$ 前端播放。
\item \textbf{检测流}: 原始帧 $\rightarrow$ YOLO-Pose推理 $\rightarrow$ NMS后处理 $\rightarrow$ \verb|DetectionResult|。
\item \textbf{姿态流}: \verb|DetectionResult| $\rightarrow$ COCO关键点规则分析 $\rightarrow$ \verb|PoseAnalysis|。
\item \textbf{事件流}: 检测结果与姿态结果 $\rightarrow$ \verb|EdgeEventAnalyzer| $\rightarrow$ \verb|SafetyEvent|。
\item \textbf{分析流}: 待复核事件 $\rightarrow$ \verb|CloudAgent| $\rightarrow$ LLM、知识库、搜索和日志查询 $\rightarrow$ \verb|CloudAnalysisResponse|。
\end{enumerate}

% ==================== 第三章 模块详细设计 ====================
\section{模块详细设计}

\subsection{共享层 (backend/shared/)}

共享层定义边端、云端和前端之间共用的数据契约、配置和运行时状态。\verb|domain/models.py|通过Pydantic定义执行目标、任务复杂度、事件等级、事件状态、姿态动作、检测框、关键点、检测结果、边端状态、调度日志、安全事件、云端分析请求/响应、事件报告和智能体问答等模型。\verb|RuntimeState|使用\verb|deque|和\verb|threading.Lock|实现线程安全状态缓存，保存最近检测结果、任务日志、事件和云端分析结果，并支持从数据库恢复历史数据。\verb|Settings|基于\verb|pydantic-settings|读取服务地址、数据库、摄像头、YOLO参数、云端Agent、LLM、搜索、Embedding、知识库目录和场所规则等配置。

\subsection{边端模块 (backend/edge\_api/)}

边端模块负责“采集—推理—分析—同步”的实时闭环。

\textbf{摄像头采集}: \verb|CameraSource|封装OpenCV摄像头读取、分辨率设置和资源释放，并提供JPEG Base64编码工具，供云端多模态分析使用。

\textbf{YOLO检测}: \verb|YoloDetector|支持\verb|.onnx|和OpenVINO \verb|.xml|两类模型。检测流程包括图像缩放、颜色空间转换、归一化、模型推理、输出转置、YOLO预测解码、NMS过滤和坐标缩放。模型元数据用于识别任务类型、类别名称和关键点名称。

\textbf{姿态分析}: \verb|PoseAnalyzer|以COCO 17关键点为输入，过滤低置信度关键点后，根据鼻尖、眼睛、耳朵、肩部、髋部、膝盖、肘部和手腕的几何关系识别低头、头部偏转、上身倾斜、举手、站立、坐下和蹲下等动作。若关键点不足或规则置信度低，则设置\verb|needs_cloud=True|。

\textbf{事件分析}: \verb|EdgeEventAnalyzer|维护短期状态，包括上一次人数、低头开始时间、聚集开始时间和事件冷却时间，将检测结果转化为结构化事件，如表\ref{tb:events}所示。

\begin{table}[H]
\caption{边端事件生成规则}
\begin{center}
\begin{tabular}{clp{7.8cm}}
\toprule
类别 & 事件类型 & 触发条件与处理方式 \\
\midrule
姿态/统计 & \texttt{person\_count} & 人数发生变化时生成信息级事件，由边端直接记录 \\
姿态/统计 & \texttt{pose\_*} & 对举手、低头、头部偏转和上身偏转等稳定姿态生成事件 \\

\textbf{编排流水线}: \verb|EdgePipeline|串联姿态分析、事件分析、任务调度和云端同步。当姿态分析需要云端复核，或边端生成\verb|cloud_pending|事件时，调度决策会升级为云端复杂任务。云端分析请求设有冷却时间，避免连续帧重复调用大模型。

\textbf{边端采集器}: \verb|EdgeCollector|在FastAPI生命周期中启动，负责采集循环、视频推流、状态发布、检测提交、事件广播和云端同步。检测推理由单线程池执行，云端同步由独立worker消费队列，因此网络抖动不会直接阻塞摄像头采集。

\subsection{云端模块 (backend/cloud\_api/)}

云端模块负责“持久化—检索—分析—报告”的智能决策闭环。\verb|CloudAgent|编排LLM、搜索、知识库和日志查询四个工具，提供自由问答、事件分析和隐患扫描三类能力。问答阶段会根据关键词自动注入历史日志摘要；事件分析阶段会结合边端证据、检测摘要、知识库命中和搜索结果生成风险结论、判断依据、处置建议和报告。

\verb|LLMClient|支持\verb|mock|、\verb|openai|和\verb|openai-compatible|三种模式，未配置外部模型时返回稳定mock结果；配置兼容接口后可调用真实模型，并支持将JPEG画面作为多模态输入。\verb|VectorKnowledgeBase|扫描知识库目录下的文本和Markdown文件，按段落切分为约500字分块；启用向量能力时写入pgvector，否则回退为关键词匹配。\verb|SearchTool|支持\verb|local|、\verb|tavily|和\verb|http-json|模式。\verb|LogQueryTool|提供历史事件查询、汇总统计和五类隐患扫描，包括未处理高风险、重复非授权进入、频繁聚集、疑似摔倒和反复超容量。

\subsection{前端模块}

系统包含两个独立前端应用。边端工作台面向实时运行状态展示，通过WebRTC接收视频，通过WebSocket接收检测结果、状态和事件，重点展示摄像头画面、检测框、人体关键点骨架、当前姿态、推理耗时、合理性检查和实时事件列表。云端控制台面向管理人员，集成云端状态、实时视频预览、事件列表、事件报告、隐患扫描、日报、智能体对话、对话历史、知识库文件列表和在线编辑等功能。

\section{总结与展望}

\subsection{项目成果}

本系统采用YOLO-Pose\cite{yolo-pose}进行人体姿态估计，基于FastAPI\cite{fastapi}和Pydantic\cite{pydantic}构建后端服务，使用WebRTC\cite{webrtc}实现实时视频传输，通过pgvector\cite{pgvector}实现向量检索增强生成(RAG)\cite{rag}，边端推理后端支持OpenVINO\cite{openvino}，整体架构体现了边云协同计算\cite{edge-cloud-collab}中“本地实时处理、云端复杂分析”的设计思想。

本项目最终形成了以下成果：

\begin{enumerate}
\item \textbf{完整的边云协同闭环}: 边端完成采集、推理、姿态分析和事件初筛，云端完成Agent分析、知识库检索、日志查询、持久化和报告生成。
\item \textbf{面向实验室场景的事件体系}: 系统结合姿态、持续时间、人数、宽高比、允许时段和容量规则生成结构化事件。
\item \textbf{可降级的智能分析能力}: 大模型、搜索和向量知识库均可在未配置外部服务时降级运行，保证课程演示稳定性。
\item \textbf{可追溯的数据管理}: 事件、分析结果和对话历史可持久化到PostgreSQL，并在服务重启后恢复到运行时状态。
\item \textbf{双前端管理界面}: 边端工作台侧重实时监测，云端控制台侧重事件管理、智能体问答、日报和知识库维护。
\item \textbf{较完整的测试体系}: 当前仓库包含54个pytest用例和集成测试脚本，覆盖核心业务模块和主要异常路径。
\end{enumerate}

\subsection{创新点}

\begin{itemize}
\item \textbf{边端规则与云端智能结合}: 将实时性要求高的姿态和事件初筛放在边端，将不确定姿态和复杂风险交给云端Agent复核。
\item \textbf{场所合理性事件建模}: 在普通视觉事件之外，引入允许时段和容量约束，使系统能够判断人员出现是否符合场所规则。
\item \textbf{日志驱动的自然语言分析}: Agent不仅回答当前画面问题，还能根据历史事件进行统计、趋势分析和隐患扫描。
\item \textbf{可降级RAG实现}: 向量检索可用时使用pgvector语义检索，不可用时回退关键词匹配。
\item \textbf{异步云端同步机制}: 边端通过独立云端同步队列上报数据，降低网络抖动对摄像头采集和本地检测的影响。
\end{itemize}

\subsection{展望}

\begin{itemize}
\item 引入门禁或预约数据，使非授权时段事件能够关联具体授权记录。
\item 引入多帧行为链分析，识别“进入实验室—长时间停留—操作设备—离开”等复杂行为模式。
\item 增加邮件、短信或企业微信告警通道，将高风险事件主动推送给管理员。
\item 为云端控制台增加趋势图、热力图和设备分组视图，提升多边端统一管理能力。
\item 完善模型配置和模型热切换机制，使不同场景可选择上半身姿态模型、摔倒检测模型或普通目标检测模型。
\end{itemize}
