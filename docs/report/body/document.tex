\section{需求分析}

\subsection{系统目标}

本项目面向机房、实验室等教学管理场景，设计并实现一套端边云协同安全监测系统。系统以边端摄像头和YOLO-Pose模型作为实时感知入口，在边缘侧完成视频采集、人体检测、人体关键点识别、姿态规则判定和初步事件生成；在云端完成事件持久化、知识库检索、大模型综合分析、日志查询、隐患扫描和日报生成；在前端通过边端工作台与云端控制台展示实时画面、检测结果、事件列表、智能体问答、知识库和报告。边缘计算相关研究强调将计算能力下沉到数据产生位置以降低时延并提高实时响应能力\upcite{shi2017edge}，这与本系统将视频推理和姿态初筛放在边端的设计目标一致。

系统设计目标不是单纯完成目标检测，而是形成“边端实时判断、云端复杂复核、管理端可视化处置”的完整业务闭环。边端优先承担实时性要求高的处理任务，云端承担需要历史上下文、知识库和大模型推理的复杂任务，从而兼顾实时性、稳定性和智能分析能力。

\subsection{非功能需求}

系统的非功能需求主要体现在实时性、边端自治、异步解耦、可扩展性和可部署性几个方面。实时性方面，边端视频推送默认限制为24fps，检测任务按\verb|EDGE_SKIP_FRAMES|跳帧执行，避免每帧推理造成CPU压力。边端自治方面，云端不可用时，边端仍能完成检测、姿态分析、事件生成和本地状态更新。异步解耦方面，摄像头采集、检测推理、视频推送、状态广播和云端同步被拆分为不同线程或异步任务，降低单个环节阻塞对整体运行的影响。可扩展性方面，LLM、搜索、知识库、数据库和推理后端均通过配置选择，便于在离线演示和真实服务之间切换。可部署性方面，系统通过Docker Compose统一编排数据库、云端API、边端API、双前端和可选边端采集器，降低课程验收环境复现难度。

\subsection{数据流设计}

系统运行时包含视频流、检测流、姿态流、事件流和分析流。视频流从摄像头原始帧开始，经缩放与编码后通过WebRTC发送到前端播放；检测流以原始帧为输入，经YOLO-Pose推理、NMS后处理和坐标缩放后形成\verb|DetectionResult|；姿态流将检测结果中的COCO关键点输入\verb|PoseAnalyzer|，得到低头、举手、站立、坐下等姿态判断；事件流由\verb|EdgeEventAnalyzer|结合检测结果、姿态结果、持续时间和场所规则生成\verb|SafetyEvent|；分析流则把待复核事件交给\verb|CloudAgent|，由LLM、知识库、搜索和日志查询共同生成\verb|CloudAnalysisResponse|。这种设计使实时链路和复杂分析链路相互独立，边端即使在云端短暂不可用时也能继续完成本地监测。

\section{模块详细设计}

\subsection{共享层设计}

共享层位于\verb|backend/shared|目录，负责定义边端、云端和前端之间共用的数据契约、配置和运行时状态。\verb|domain/models.py|通过Pydantic定义执行目标、任务复杂度、事件等级、事件状态、姿态动作、检测框、关键点、检测结果、边端状态、调度日志、安全事件、云端分析请求/响应、事件报告和智能体问答等模型。\verb|RuntimeState|使用\verb|deque|和\verb|threading.Lock|实现线程安全状态缓存，保存最近检测结果、任务日志、事件和云端分析结果，并支持从数据库恢复历史数据。\verb|Settings|基于\verb|pydantic-settings|读取服务地址、数据库、摄像头、YOLO参数、云端Agent、LLM、搜索、Embedding、知识库目录和场所规则等配置。

\subsection{边端模块设计}

边端模块位于\verb|backend/edge_api|目录，负责“采集—推理—分析—同步”的实时闭环。\verb|CameraSource|封装OpenCV摄像头读取、分辨率设置和资源释放，并提供JPEG Base64编码工具，供云端多模态分析使用。\verb|YoloDetector|支持\verb|.onnx|和OpenVINO \verb|.xml|两类模型，检测流程包括图像缩放、颜色空间转换、归一化、模型推理、输出转置、YOLO预测解码、NMS过滤和坐标缩放。模型元数据用于识别任务类型、类别名称和关键点名称。

人体姿态估计是系统事件生成的基础。相关研究通常将人体姿态估计理解为对人体关节点或关键部位的位置预测，并进一步服务于行为理解和场景分析\upcite{pose2021survey}。本系统中的\verb|PoseAnalyzer|以COCO 17关键点为输入，过滤低置信度关键点后，根据鼻尖、眼睛、耳朵、肩部、髋部、膝盖、肘部和手腕的几何关系识别低头、头部偏转、上身倾斜、举手、站立、坐下和蹲下等动作。若关键点不足或规则置信度低，则设置\verb|needs_cloud=True|，交由云端复核。

\verb|EdgeEventAnalyzer|维护短期状态，包括上一次人数、低头开始时间、聚集开始时间和事件冷却时间，将检测结果转化为结构化事件，如表\ref{tb:events}所示。\verb|EdgePipeline|进一步串联姿态分析、事件分析、任务调度和云端同步。当姿态分析需要云端复核，或边端生成\verb|cloud_pending|事件时，调度决策会升级为云端复杂任务。云端分析请求设有冷却时间，避免连续帧重复调用大模型。\verb|EdgeCollector|在FastAPI生命周期中启动，负责采集循环、视频推流、状态发布、检测提交、事件广播和云端同步。检测推理由单线程池执行，云端同步由独立worker消费队列，因此网络抖动不会直接阻塞摄像头采集。

\subsection{云端模块设计}

云端模块位于\verb|backend/cloud_api|目录，负责“持久化—检索—分析—报告”的智能决策闭环。\verb|CloudAgent|编排LLM、搜索、知识库和日志查询四个工具，提供自由问答、事件分析和隐患扫描三类能力。问答阶段会根据关键词自动注入历史日志摘要；事件分析阶段会结合边端证据、检测摘要、知识库命中和搜索结果生成风险结论、判断依据、处置建议和报告。大模型检索增强生成的核心思想是在生成前引入外部知识检索，以缓解知识过时和事实依据不足等问题\upcite{gao2023rag}，本系统将这一思想用于实验室安全规范、历史事件和处置建议的联合分析。

\verb|LLMClient|支持\verb|mock|、\verb|openai|和\verb|openai-compatible|三种模式，未配置外部模型时返回稳定mock结果；配置兼容接口后可调用真实模型，并支持将JPEG画面作为多模态输入。\verb|VectorKnowledgeBase|扫描知识库目录下的文本和Markdown文件，按段落切分为约500字分块；启用向量能力时写入pgvector，否则回退为关键词匹配。\verb|SearchTool|支持\verb|local|、\verb|tavily|和\verb|http-json|模式。\verb|LogQueryTool|提供历史事件查询、汇总统计和五类隐患扫描，包括未处理高风险、重复非授权进入、频繁聚集、疑似摔倒和反复超容量。

\subsection{前端模块设计}

系统包含两个独立前端应用。边端工作台面向实时运行状态展示，通过WebRTC接收视频，通过WebSocket接收检测结果、状态和事件，重点展示摄像头画面、检测框、人体关键点骨架、当前姿态、推理耗时、合理性检查和实时事件列表。云端控制台面向管理人员，集成云端状态、实时视频预览、事件列表、事件报告、隐患扫描、日报、智能体对话、对话历史、知识库文件列表和在线编辑等功能。报告导出采用“后端结构化报告 + 前端下载/打印”的方式：日报可下载Markdown，并可打开HTML日报页面后调用浏览器打印生成PDF；智能体检测分析结果可构造Markdown报告下载，也可生成独立HTML页面并调用打印功能导出PDF。

\subsection{WebSocket消息格式}

边端和云端通过WebSocket向前端推送实时状态，消息采用统一的\verb|type + data|结构。检测结果使用\verb|detection|类型，任务日志使用\verb|task_log|类型，安全事件使用\verb|event|类型，云端分析结果使用\verb|analysis_result|类型，边端状态使用\verb|status|类型。统一消息格式使前端能够以较少的分支逻辑处理不同业务数据，同时保留扩展新消息类型的空间。

\begin{lstlisting}[language={}]
{ "type": "detection",       "data": DetectionResult }
{ "type": "task_log",        "data": TaskLog }
{ "type": "event",           "data": SafetyEvent }
{ "type": "analysis_result", "data": CloudAnalysisResponse }
{ "type": "status",          "data": EdgeStatus }
\end{lstlisting}

\subsection{核心数据模型}

% ==================== 第八章 总结与展望 ====================
\section{总结与展望}

\subsection{项目成果}

本项目最终实现了一个较完整的端边云协同安全监测原型。边端已经能够完成摄像头采集、YOLO-Pose推理、人体关键点解析、姿态规则分析、边端事件初筛和云端异步同步；云端已经能够完成事件持久化、智能体分析、知识库检索、日志查询、隐患扫描、对话历史恢复和报告生成；前端已经能够展示实时视频、检测框、关键点骨架、事件列表、分析结果、日报、智能体对话和知识库内容。系统既支持Markdown文本报告，也支持日报和智能体检测分析报告的PDF导出，便于课程验收时展示和提交。

与普通单机视觉检测程序相比，本项目的成果在于形成了从实时感知到风险判断再到报告输出的闭环。边端规则用于保证实时性和离线可用性，云端Agent用于补充历史上下文和领域知识，数据库用于保证事件、分析和对话可追溯，双前端用于区分边端实时监测和云端综合管理。整体设计体现了边缘计算“靠近数据源处理实时任务”和云端“集中管理复杂任务”的协同思想\upcite{shi2017edge,chen2018edge}。

\subsection{创新点}

本系统的第一个创新点是将边端规则与云端智能分析结合。姿态和人数变化等实时性要求高的任务在边端直接处理，不确定姿态、疑似摔倒、频繁聚集等复杂风险再交给云端Agent复核。第二个创新点是引入场所合理性事件建模，在普通视觉事件之外加入允许时段和容量约束，使系统能够判断人员出现是否符合实验室管理规则。第三个创新点是日志驱动的自然语言分析，Agent不仅回答当前画面问题，还能根据历史事件进行统计、趋势分析和隐患扫描。第四个创新点是可降级RAG实现，向量检索可用时使用pgvector语义检索，不可用时回退关键词匹配。第五个创新点是面向验收展示的报告导出，日报和检测分析报告既保留Markdown文本，也可通过HTML页面与浏览器打印机制导出PDF。第六个创新点是异步云端同步机制，边端通过独立队列上报数据，降低网络抖动对摄像头采集和本地检测的影响。

\subsection{展望}

后续可以从数据接入、行为理解、告警通道和多设备管理几个方向继续完善。数据接入方面，可以引入门禁或预约数据，使非授权时段事件能够关联具体授权记录。行为理解方面，可以引入多帧行为链分析，识别“进入实验室—长时间停留—操作设备—离开”等复杂行为模式。告警通道方面，可以增加邮件、短信或企业微信通知，将高风险事件主动推送给管理员。多设备管理方面，可以为云端控制台增加趋势图、热力图和设备分组视图，提升多个边端节点统一管理能力。模型管理方面，可以继续完善模型配置和模型热切换机制，使不同场景可选择上半身姿态模型、摔倒检测模型或普通目标检测模型。
