\section{系统演示场景}

\begin{enumerate}
\item 启动系统（Docker Compose或本地启动）
\item 摄像头前依次展示正常姿态、低头、多人进入等场景
\item 边端工作台实时显示检测框、关键点骨架和事件告警
\item 云端控制台同步显示事件列表、风险等级和Agent分析结果
\item 展示合理性分析：非开放时段检测触发unauthorized\_time事件
\item 管理员输入"分析今天的异常事件"，Agent输出汇总报告
\item 查看每日检测报告，导出Markdown日报
\end{enumerate}

\section{AI使用说明}

本项目在设计和实现过程中使用了AI工具进行辅助，属于由本人主导、AI辅助完成的课程设计项目。具体辅助内容包括：在已经完成YOLO-Pose人体姿态检测的基础上，AI辅助比较了智能健身、课堂行为识别、机房/实验室安全监测等不同应用方向，并协助将项目功能收束为端边云协同安全监测场景；在系统设计阶段，AI辅助梳理了“摄像头采集---边端推理---姿态规则分析---事件生成---云端复核---报告导出”的数据流；在工程实现阶段，AI围绕FastAPI边端服务、WebRTC与WebSocket实时推送、PostgreSQL事件持久化、知识库检索、CloudAgent事件分析和日报生成等模块，提供了接口组织、数据模型命名、测试覆盖方向和错误排查思路；在文档与展示材料阶段，AI辅助整理了架构描述、截图说明、LaTeX排版和项目海报中的文字表达。

项目的需求理解、应用场景取舍、代码集成、功能联调和最终验收内容均由本人负责完成。本人完成了YOLO模型接入、摄像头采集与推理后处理、关键点坐标还原与骨架绘制、姿态规则调参、事件冷却与合理性检查逻辑、云端API和数据库联调、前端页面展示、Docker Compose部署、实际运行截图采集以及演示流程设计等工作。AI输出没有直接替代本人对系统架构、核心代码、测试结果和项目结论的判断，所有建议均经过人工筛选、修改、运行验证和取舍，最终提交内容以本人实际实现和调试结果为准。

\section{项目源码结构}

\begin{lstlisting}[language=bash]
src/backend/
  shared/           # 共享层 (models/config/state/scheduler)
  cloud_api/        # 云端服务 (routes/cloud)
  edge_api/         # 边端服务 (routes/runtime)
src/frontend/
  cloud_frontend/   # 云端控制台 (Vue3, 5页面)
  edge_frontend/    # 边端工作台 (Vue3, WebRTC)
tests/              # 54个测试用例
docs/               # 设计文档
data/knowledge/     # 知识库文件(3份)
scripts/            # 集成测试/模型下载
docker-compose.yml  # 6服务编排
\end{lstlisting}
